实体链接技术研究

Posted on 2016-08-17

实体链指的主要目标是识别文本中名称指代哪个现实世界中的实体。具体而言,实体链指是将给定文本中的个名称字符串映射到知识库中的相应实体上去,如果知识库尚未收录相应实体,则返回空标记。近些年,美国国家 标准技术研究所(NIST)举办了多次以实体链指为核心的国际评测任务。实体 链指在信息抽取、问答系统、机器翻译、信息检索等多个自然语言任务中都有着广泛的应用。

实体链接的定义

  • $\mathbb{E}$: 所有实体的集合
  • $\mathbb{K}$: $\mathbb{E}$的一个子集,称作知识库
  • $\mathbb{D}$: 文档集
  • q: 实体链指的输入, 两部分构成：查询文档$\mathcal{D}(\mathcal{D} \in \mathbb{D})(Query\ Document)$、查询名称$m\ (m \subset \mathcal{D})(mention\ name)$
  • e: m所对应的目标实体

实体链接面临的问题

一义多词，一词多义的问题

实体链接的步骤

  • 实体抽取: 实体发现(entity discovery)，实体识别(entity recognization)
    • 实体类别(人名、地名、组织名) name of book, movie, music, cuisine，fauna, herb
    • 无论是TAC还是NLPCC举办的实体链接任务，实体抽取（实体发现）都已经做了
  • 实体候选集生成:
    • 生成的实体候选集圈定了候选实体的范围
    • 消歧在此范围内选择最适合上下文的实体作为链指结果
    • 候选实体集合的质量与两个因素有关:
      1. 是否包含目标实体
      2. 候选实体的数目
    • 候选生成的主要目标:
      1. 包含目标实体
      2. 集合越小越好
  • 实体消歧
    • 从候选集合中找到最适合上下文的实体作为链指结果。

短文本实体链接

文本噪音
  1. 错别字(misspell)
    • 中文：同音异形(拼音距离，模糊匹配)
      • 孙杨->孙扬
    • 英文：字母错误，字母交换，间隔符(jaccard distance, edit distance, dice index)
      • micheal->michear
      • from->form
      • countryside->country side
  2. 简称(short name)
    • rule1:前两个词的首字符
      • 北大->北京大学
    • rule2:第一个词的首字符以及第二个词
      • 中移动->中国移动通信公司
    • rule3:前两个词
      • 香港迪士尼->香港迪士尼乐园

        不全，总之简称包含于全称中
        $$Sim(n_{e},n_{q})=\frac{\sum\nolimits_{w\in n_{q}}Len(LCS(w,n_{e}))}{\sum\nolimits_{w\in n_{q}}Len(w)}$$

  3. 别名(alias)
    • 别名与正式名称字面相差很大
    • 使用外部搜索引擎
候选集生成
  • raw query: 在知识库中直接搜索m
  • 扩展多个名称：
    • 借助外部搜索引擎(m+维基百科)
    • 借助简称形成的规则
  • 借助字面相似度
    $$jaccard(A, B) = \frac{|A \bigcap B|}{ |A\bigcup B|}$$
    $$Dice(A, B) = \frac{2|A \bigcap B|}{ |A|+ |B|}$$
  • 借助拼音距离

    平舌音->翘舌音
    前鼻音->后鼻音
    l->n

  • 带权重
    给各种方法加上权重，设置thredhold，缩小候选集大小
  • 以上方法的组合
实体消歧
  • 基于TF-IDF值计算相似度
    $$tf-idf{t,d} = (1 +\log tf{t,d}) \cdot \log \frac{N}{df_t}$$
    1. 利用TFIDF值提取microblog的特征词
      1. 候选实体的描述文字踹出现的特征词的tfidf相加作为该候选实体的相似度
    2. 取相似度最大的候选实体
  • 基于word2vec计算相似度:把词汇向量化之后，计算microblog特征词与候选实体特征词的相关度
  • 基于标签(主题)
  • 图：出入度大小
  • 流行度
    • 实体被浏览的次数
    • 描述实体文本的长度
    • 被其他实体描述页面引用的次数
  • 以上各种方法的组合，加上权重，设置thredhold

mac读写移动介质

在新买的mac上插优盘或者移动硬盘会发现插入的介质只能读不能写，这是因为mac的文件系统是HFS(hierarchical file system, which is mac standard file system)，大多数移动介质是NTFS(NT file system, 这是windows的标准文件系统)。 不巧的是，HFS可以读NTFS，但不能写。mac不能写优盘这让人好生抓狂，下面介绍一种解决方法。 Free!

利用Github和hexo搭建博客

Posted on 2016-08-01

搭建博客主要分两部分，一部分是配置Github端，一部分是配置本地的hexo
先讲Github端的配置

Github端配置

  • 创建博客仓库
    • 登录或者注册Github，假设你Github的用户名为yourname
    • 新建一个仓库，命名为yournamename.github.io
    • 完成后打开该仓库，点击setting，选择Automatic page generator
    • 最后在浏览器的地址栏输入yourname.github.io即可看到自己的博客
    • 至此，博客仓库创建完毕

  • 安装git

    • linux和mac用户输入git，按提示安装git
    • windows用户下载git

  • 配置ssh
    windows用户打开git bash, linux和mac打开终端

    • 输入cd ~/.ssh && ssh-keygen 这一步生成密钥，一路按回车即可
    • 复制id_ras.pub到剪切板
      • On OS X run: cat id_rsa.pub | pbcopy
      • Linux run: cat id_rsa.pub | xclip
      • On Windows (via Cygwin/Git Bash) run: cat id_rsa.pub | clip
    • 打开github的ssh设置页面
      • 点击新建ssh key
      • 把剪切板的内容复制到key栏中（title栏自便）
      • 点击Add SSH key（出于安全考虑Github会让输入密码）
    • 至此ssh配置完毕
  • 配置本地git
    • git config –global user.name “yourname”
    • git config –global user.email “youremail”
  • 至此Github端配置完毕
    下面讲述hexo的配置过程

    Hexo配置

  • 下载并安装node.js
  • 在终端中输入node -v以及npm -v如果出现版本信息，则nodejs正确安装，出错了自行搜索

  • cd到合适的目录下输入以下命令安装hexo

  • 输入hexo -v 查看hexo的版本信息，如果正确显示，则意味着这hexo正确安装

  • 然后输入 hexo init初始化Hexo（应该是clone远程仓库到本地）
  • hexo g生成相关文件
  • hexo s启动本地服务器，这时在本地用浏览器打开localhost:4000查看效果
